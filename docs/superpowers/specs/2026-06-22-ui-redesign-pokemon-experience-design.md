# 設定UX刷新：ポケモン体験ベースの再設計 設計仕様

**作成日**: 2026-06-22
**対象**: `pokefollower-desktop`（設定ウィンドウのUX）
**ステータス**: ブレインストーミング合意済み → ユーザーレビュー待ち

## ゴール

「とりあえず実装してみた機能」が1枚のカードに散らかった現状の設定ウィンドウを、**本家ポケモンの情報設計（手持ち／ボックス／サマリー）に寄せた体験**へ作り変える。普段使う最小限を表に出し、煮詰まっていない機能は `Coming Soon` として隠して後でちゃんと作る。命名は英語ジャーゴンを全廃し日常語へ。

非ゴール：マスコット本体の追従挙動の変更。新しい行動モードの実装（おさんぽ等は今回 Coming Soon）。

## Global Constraints（不変条件）

- **状態機械を変更しない**: `src/main/follower-sim.js`、`crates/follower_core/**`、`native/*.wasm` は触らない。今回は設定ウィンドウ（レンダラ）のUX刷新に限定。
- **Coming Soon の機能はいったんOFF**: 端休み・カーソルよける・アプリ反応等は**設定の既定値を無効化**して素直な「ついてくる」だけにする。状態機械コードは無変更（既定値だけ変える）。機能コードは消さない（後で本気版に差し替える土台）。
- **vanilla HTML/CSS 継続**: 既存レンダラの流儀（`settings.html` 内CSS・ビルド不要）を踏襲。Tailwind等は導入しない。タブ切替は軽量JSのみ。
- **スプライト描画の流用**: あいぼうの相棒大表示は、オーバーレイ（`src/overlay/overlay.js`）のスプライトシート描画ロジック（frame/row/AnimData）を設定レンダラ側に流用してIdleアニメを描く。新しいアニメ機構は作らない。
- **既存ロジックは流用**: パック読み込み（`pack-reader.js`）・index・フィルタ純関数（`src/settings/filter.mjs`）・検索エンジン（`search-engine.js`）は活かす。作り直すのはUI層。
- **PRベース**: `feature/ui-redesign` を現行main基準で切る。Nicolas churn 対策で着手前に `git fetch && merge --ff-only origin/main`。
- **検証ゲート維持**: `npm run verify:local` + Vitest を緑に保つ。設定UIの検証（`verify-settings-ui.cjs`）は新DOM構造に合わせて更新する。

## アーキテクチャ概要

下タブ **3画面**（あいぼう / ボックス / せってい）。マスコット選択・追従調整は既存データ／IPCにそのまま乗る。追加するのは (1) 全1010体のタイプデータ生成、(2) タイプ色パレット、(3) 3タブUI（DOM/CSS/JS刷新）、(4) ニックネーム保存、(5) 手持ち最大6・先頭=相棒モデル、(6) 検証更新。

## 画面構成

### 1. あいぼう（ホーム＋手持ちを統合）

本家の「パーティ＋サマリー」を1画面に。相棒（手持ちの先頭）を大きく見せつつ、手持ち6体を同画面に常駐させる。

- **相棒の大きな表示**: 実際のマスコット（PMDドット絵）の **Idle アニメを大きく**表示（背景なし）。デスクトップにいる相棒と同じ絵＝WYSIWYG・生きてる感。既存パックのスプライトシート（Idle）を流用＝新規アセット不要。ボックスのタイルは現状（pokemondb系）のまま＝画面間で絵柄は多少混ざるが許容。
- **見出し**: `#025` 図鑑番号 / **タイプチップ**（タイプ色付き、例 でんき=黄）/ **ニックネーム ✎**（未設定時はデフォルト日本語名）
- **スライダー3つ**: `大きさ`（0.5〜5.0x）/ `距離`（0〜100）/ `速さ`
- **手持ち列（最大6枠）**: 先頭＝相棒（ハイライト）。**枠タップで先頭へ＝即デスクトップの相棒が替わる**。各枠は `×` で手持ちから外す。空き枠は `＋`（押すとボックスへ）。※ニックネーム編集は相棒見出しの ✎ のみ（枠では行わない）
- **`おさんぽ（近日）`** の無効ピル（ロードマップを匂わせる。押すと「準備中」表示）

### 2. ボックス（全種から探す）

Pokémon HOME のボックス的なグリッド。

- **手持ち列を上部に常駐**（小さく）: あいぼうと同じ6枠を表示。「どこに入るか」が常に見える＝ダイアログ/ドラッグ不要の鍵。
- **検索**（`名前・タイプ・作品名で検索`）: 既存 `search-engine.js` を使用。**タイプデータ追加で全1010体がタイプ検索可能に**（現状12体→1010体）
- **種類セレクタ**: `通常 ⇄ 地方フォルム`（既存 `kind` フィルタ流用）
- **フィルタチップ**: `通常`時=世代（全/赤緑〜SV）、`地方`時=地方（全/アローラ/ガラル/ヒスイ/パルデア）。加えて **タイプ絞り込み**（HOME的。タイプ色チップ）
- **グリッド**: 全1010体。各タイルに #番号・名前・**タイプ色の縁**。手持ちに入っている子は★等で既属表示。
- **タップ挙動（確定）**:
  - 手持ちに**空きがあれば即追加**（確認なし）
  - **満杯（6）なら、上部の手持ち列が点滅し「入れ替える子をタップ」**→タップした枠を置換。余白タップで中止。
  - ダイアログ・ドラッグ&ドロップは使わない（小窓で精密操作がつらい/実装過剰なため）。

### 3. せってい（細かい調整＋ロードマップ）

- **表示する**（ON/OFF。現 `Enable Follower`）
- **通知コンパニオン**（+ `TEST`）
- **Codex pet 書き出し**（+ `EXPORT`）
- **カーソルをよける**（既定ON・よけ方 ふつう/つよい）と **アプリに合わせる**（opt-in）のトグル（Nicolas 実装を採用）
- **近日公開**セクション（無効リスト・説明付き）: おさんぽ / せいかく / 作業見守り / はじっこで休む

## データモデル

### 新規：タイプデータ `assets/packs/type-data.json`

- 形: `{ "<dexまたはpackキー>": { "types": ["electric"] } }`（**タイプのみ**。図鑑の分類/高さ/重さ/フレーバー説明は持たない）
- **図鑑説明を持たない理由**: フレーバー文は作品（HGSS/RS/剣盾…）ごとにテキストが異なり「どの版を正とするか」問題が出る。今回の体験では不要と判断（将来やるなら別spec）。
- **キー設計**: 通常種は dex 番号。**地方フォルムは原種とタイプが異なる**ため、フォルムは pack id（`retro/forms/alola/026-raichu`）でキーする（アローラライチュウ=でんき/エスパー）。
- **生成**: アセット生成パイプライン（`scripts/gen-*.mjs` の流儀）で PokéAPI 等から build 時に取得し、スナップショットJSONをコミット。出典・ライセンスを明記。状態機械・既存アセットは無変更。
- **タイプ色パレット**: 18タイプの標準カラーをコード内に内蔵（`src/settings/type-colors.mjs`）。`search-metadata.json` の `facets.types`（ja名・aliases）と整合させ、未定義タイプ（現状12→18）を補完。

### 既存の拡張

- **ニックネーム**: `settings-store.js` に `nicknames: { "<packId>": "<name>" }` を追加（ローカル保存）。あいぼうの ✎ で編集。未設定はデフォルト日本語名にフォールバック。
- **手持ち**: 既存 favorites（最大12）を**最大6**へ。**先頭＝相棒（デスクトップ表示）**。移行時は既存favoritesの先頭6体を引き継ぐ。自動交代（rotation）は廃止（コードは設定の Coming Soon 裏に温存可）。
- **pack-reader / index**: 既存の `region`/`ja` に加え、読み出し時に**タイプデータ（types）**をマージして返す（UIが1回で使える形に）。

## 実装後のポリッシュと根拠（2026-06-22 起動検証で確定）

実機（`npm start`）で見ながらユーザーと詰めた、見た目の決定とその理由。**なぜこの実装にしたか**の記録。

### なぜ「ポケモン体験ベースの3タブ」なのか（大方針）
- 旧UIは機能を1枚のカードに縦積みした“設定ダンプ”で、カジュアルに選びたい人もコア調整したい人も同じ壁を見ていた。
- マスコット＝「今日連れて歩く相棒を手持ちから選ぶ」行為。これは本家ポケモンの **手持ち/ボックス/サマリー** の情報設計とほぼ一致する。だから `あいぼう（サマリー＋手持ち統合）/ ボックス（全種）/ せってい` の3分割にした。
- 「ホーム」と「手持ち」を別タブにすると先頭=相棒で食い合うため、本家のパーティ画面に倣い **あいぼうに統合**（相棒を大きく見せつつ手持ち6体を同画面に常駐）。

### なぜ「半端な機能はComing Soon＝既定OFF」なのか
- おさんぽ・せいかく・作業見守り・端休み・よける・アプリ反応は「とりあえず実装」段階でUXが練られていなかった。綺麗なUIに半端を並べるのは本末転倒。
- よって**UIから隠し、既定値をOFF**にして素直な「ついてくる」だけに。コードは温存し、作り直してから出す。状態機械は無変更（既定値だけ変更）。

### なぜ「タイプ（＋色）を全1010体」入れたのか
- タイプはポケモンの識別の核。色テーマ・絞り込み・検索の土台になる。
- 既存の自然言語検索は12体しかタイプを持たず看板倒れだった。`type-data.json`（PokéAPIスナップショット）で全1010体に拡張し、検索窓で「でんき」「water」が全電気タイプにヒットするようにした（pack.types を検索インデックスにEN+JAで畳み込み）。
- 図鑑の**説明文は不採用**：作品（HGSS/RS/剣盾…）でテキストが違い「どの版が正か」問題が出るため。タイプのみ。

### 相棒スプライトの「大きさ」：整数倍＋上限
- ドット絵は元が小さく(～24〜64px)、倍率を上げすぎると1ドットが粗くなって**逆に見づらい**。
- そこで枠に収まる**整数倍**（nearest=くっきり）で拡大し、**上限4倍**で「大きいけど粗くない」所で止めた。

### 相棒の「中央化」：実測ベースの固定オフセット
- 中央寄せでも相棒が**上に**見えた。原因は **Idleコマの中でキャラが上寄り**なこと。
- 6種で実測したところ、コマ内のキャラ中心が **cy≈0.31〜0.40（平均0.35）= 一貫して上寄り**（下35%は影/接地の空白）。全種でほぼ一定だったので、**コマ高さの0.15ぶん下げる固定オフセット**で全種を中央化できた（per-sprite解析は不要）。
- 実行時の画素解析トリミングは不可：設定窓は `file://`、スプライトは `app://` で**canvasがtaintする**＋CSP `connect-src 'none'`。よってオフセット方式を採用。

### 手持ち/ボックスの「絵柄の統一」：ビルド時に画像正規化
- フォルムのUIタイルは全部 96×96 の正方形だが、**コマ内のキャラの位置・大きさが個体ごとにバラバラ**で、`object-fit`/`object-position` では揃わない（正方形ゆえCSSでは動かせない）。
- 根本対応として **ビルド時に画像を正規化**（`scripts/normalize-ui-tiles.mjs`：透明余白トリム→96×96中央フィット）。フォルム54枚の中身を**中央(0.49,0.49)・均一サイズ**に揃えた。
- 通常種(pokemondb)は**実サイズを反映**して占有率がバラバラ（ピカチュウ41%〜ゲンシ系98%）。正規化フォルムを100%のまま並べると通常種より大きく見えたので、**ボックス/手持ちではフォルムだけ見た目を縮小**（枠は固定＝名前/番号の並びは崩さない）して馴染ませた。

### その他の手触り
- **相棒切替の二重チラつき**：`getPackMeta` 待ちの間に旧スプライトが残るのが原因。await前に旧表示を即クリア＋切替レースガード、初期フレームを同期表示で解消。
- **あだ名**：Electronは `window.prompt` 非対応。`heroName` をその場でインライン入力に切替（Enter保存/Esc取消/空で解除）。
- **手持ちの×ボタン見切れ**：スロットの `overflow:hidden` と行の `overflow-x:auto` がクリップしていたので解除し前面化。
